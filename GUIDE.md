# GUIDE — установка Воси (Vosmenog)

Пошаговая установка дисциплинированного агента. Базовый сценарий — **локальная
машина**; серверный (VPS) — в конце, надстройкой. Гайд обкатан вживую: грабли,
которые реально встречаются, помечены и обойдены.

> **Рабочая среда (проверено):** opencode-движок + **VSCode со встроенным терминалом**
> на компе, и **Termius** для доступа с телефона по SSH. Отдельный TUI-клиент opencode
> на практике не нужен — работаем через терминал VSCode.

---

## 0. Что ставим

Дисциплинированный агент-исполнитель для [opencode](https://opencode.ai). Три слоя:
- **METHOD** — метод работы (грузится всегда);
- **Vosmenog** — режим-исполнитель с урезанными правами (таб-переключаемый);
- **память + шаблоны** (эпик/задача/баг).

Нужен Unix-терминал: на Mac/Linux он родной, на Windows — через WSL.

---

## Шаг 0 — Linux (для Windows)

WSL — это настоящий Linux внутри Windows. В **PowerShell от администратора**:
```
wsl --install
```
Перезагрузись. После перезагрузки сам откроется терминал Ubuntu и попросит завести
пользователя.

> **Грабля:** имя пользователя — **только строчными буквами** (`skunokil`, не
> `Skunokil`), иначе Linux его отвергнет. Пароль печатается «вслепую» (символы не
> видны) — это нормально, набирай и жми Enter.

Дальше **все команды — в терминале Ubuntu**, не в PowerShell. Открыть Ubuntu потом:
Пуск → набрать `Ubuntu` → запустить (оранжевый значок).

**Mac/Linux:** терминал уже Unix, WSL не нужен — пропускай шаг 0.

## Шаг 1 — Базовые инструменты

В свежей Ubuntu может не быть нужного. Поставим разом:
```
sudo apt update
sudo apt install -y curl unzip git
```

## Шаг 2 — opencode

```
curl -fsSL https://opencode.ai/install | bash
```
Проверка версии:
```
opencode --version
```

> **Грабля (частая):** `--version` показывает **старую** версию или «command not
> found». Две причины:
> 1. shell ещё не подхватил новый путь → `source ~/.bashrc` и повтори, или закрой/
>    открой терминал заново.
> 2. **opencode уже стоял на Windows** (через npm) — тогда WSL подмешивает виндовый
>    PATH, и Linux зовёт старый виндовый бинарник. Диагностика:
>    ```
>    which -a opencode
>    ```
>    Если первым идёт путь `/mnt/c/...` — это виндовый. Поставь линуксовый вперёд:
>    ```
>    echo 'export PATH="$HOME/.opencode/bin:$PATH"' >> ~/.bashrc
>    source ~/.bashrc
>    opencode --version
>    ```
>    `which opencode` теперь должен указывать в `/home/...`, а версия — свежая.

## Шаг 3 — Получить комплект

Репозиторий публичный — клонируется без авторизации:
```
git clone https://github.com/Skunokil/vosmenog.git
cd vosmenog
```

---

## Шаг 4 — Установщик

```
chmod +x setup.sh
./setup.sh
```
Скрипт разложит файлы, подключит метод, заведёт память и лаунчер `go`. Существующий
конфиг **бэкапит**, память (`journal.md`) не трогает.

---

## Шаг 5 — Периметр (безопасность)

Установщик спросит каталоги, которые агент **не должен трогать** (прод, секреты).
Это физический барьер — агент не дотянется, даже если захочет.

> **Грабля:** вводи **только пути**, через пробел, чистой строкой
> (`~/bot_factory ~/.ssh`). Не вставляй лишний текст — скрипт запишет его как есть.
> На чистой локалке закрывать нечего → нажми **Enter**, пропусти.

---

## Шаг 6 — Активация и проверка

```
source ~/.bashrc
opencode agent list
```
В списке должен быть **Vosmenog (all)** с урезанными правами (bash под ask, prod
закрыт). Затем:
```
opencode
```
Нажми **Tab** — режимы переключаются: `build`, `plan`, **Vosmenog**. Выбери Vosmenog,
напиши `привет`. Должен ответить быстро и в характере — представиться исполнителем,
потребовать оформить EPIC/TASK.

> **Грабля (если старт виснет надолго):** старые установки несли в конфиге блок
> `references.memory`, который вешал слабые модели на ритуале памяти при старте.
> В текущем репо это уже исправлено. Если ставил из старой версии и старт виснет —
> убери `references` из `~/.config/opencode/opencode.json`.

---

## Шаг 7 — Лаунчер проектов

`go` заводит проект и запускает opencode прямо в его папке (чтобы подхватились слоты
`<проект>/AGENTS.md`). Открой `~/.bashrc`, найди функцию `go`, добавь свой проект:
```bash
go() {
  case "$1" in
    myproj) cd ~/path/to/myproj && opencode ;;
    *) echo "go: неизвестный проект '$1'" ; return 1 ;;
  esac
}
```
`source ~/.bashrc` — и `go myproj` готов.

---

## Шаг 8 — Новый проект под Восю

1. В папке проекта создай `AGENTS.md` из шаблона
   `~/agent-os/project-slots.template.md`, заполни слоты (стек, пути, патч, билд,
   git-состояние, гейты). `AGENTS.md` — родной формат opencode, подхватывается сам.
2. Переключись на режим Vosmenog (Tab).
3. Цикл: **обсуждение → агент пишет EPIC → PBR → go → задачи → resolve.**
   Подробно — `~/agent-os/ONBOARDING.md`.

---

## Шаг 9 — Первый эпик (обкатка)

Возьми маленькую реальную задачу. В режиме Vosmenog опиши цель, обсудите развилки;
агент оформит EPIC (DoR/DoD/AC) — проверь, что развилки закрыты; дай go, агент
раздробит на задачи и исполнит под подтверждение; в конце — resolve.

---

## Серверный сценарий (VPS)

То же самое, но opencode и проекты живут на сервере, а ты подключаешься: с компа —
VSCode «Remote-SSH», с телефона — Termius по SSH. Отличия от локалки:

1. **opencode на сервере уже есть** — шаг 2 пропускаешь.
2. **Периметр обязателен.** На сервере есть прод и секреты — закрой их наглухо.
   На шаге 5 вводи реальные пути, например: `~/bot_factory ~/.ssh`.
3. **Если на сервере была ручная/старая установка** — перед `setup.sh` сделай бэкап
   и убери устаревшее:
   ```
   cp -r ~/agent-os ~/agent-os.bak-$(date +%F) 2>/dev/null
   rm -f ~/agent-os/TZ.template.md      # старое имя шаблона
   ```
   `setup.sh` сольёт конфиг бережно (бэкапит, не задвоит instructions), память
   сохранит.
4. Установка: `git clone https://github.com/Skunokil/vosmenog.git && cd vosmenog &&
   ./setup.sh`, на периметре — пути прода, дальше как на локалке.
