# GUIDE — установка Воси (Vosmenog)

Пошаговая установка дисциплинированного агента на **локальную машину**.
Серверный сценарий (VPS, прод) — надстройка, см. последний раздел.

> Лучший проверенный сценарий работы: **opencode-движок + VSCode со встроенным
> терминалом** на компе, и **Termius** для доступа с телефона по SSH. Отдельный
> TUI-клиент opencode на практике оказался не нужен — работаем через терминал VSCode.

---

## 0. Что мы ставим

Три слоя, которые вместе дают агента, который не уходит в самодеятельность:
- **METHOD** — метод работы (always-context, грузится всегда);
- **Vosmenog** — режим-исполнитель с урезанными правами (таб-переключаемый);
- **память + шаблоны** (эпик/задача/баг).

Базовый сценарий — **локальный**. Нужен Unix-терминал: на Mac/Linux он родной,
на Windows — через WSL (настоящий Linux внутри Windows).

---

## 1. Среда

### Windows → поставить WSL
WSL — это официальный Linux внутри Windows. В PowerShell от администратора:
```
wsl --install
```
Перезагрузись, заведи пользователя Ubuntu. Дальше **все команды — в терминале Ubuntu**,
не в PowerShell.

### Mac / Linux
Терминал уже есть. Идём дальше.

### VSCode + терминал
Поставь VSCode, расширение для подключения (на Windows — «WSL», для сервера позже —
«Remote-SSH»). Работать будешь во встроенном терминале VSCode (`Ctrl+~`).

---

## 2. Установка opencode
По официальной инструкции opencode для твоей ОС (в WSL — как для Linux).
Проверка:
```
opencode --version
```
Версия в ответ — значит стоит.

---

## 3. Получить комплект
```
git clone <YOUR_REPO_URL> vosmenog
cd vosmenog
```
> `<YOUR_REPO_URL>` — адрес публичного репозитория Воси. Если репо ещё нет —
> скопируй папку `vosmenog/` на машину любым способом и зайди в неё.

---

## 4. Запуск установщика
```
chmod +x setup.sh
./setup.sh
```
Скрипт разложит файлы по местам, подключит метод в always-context, заведёт память
и лаунчер `go`. Существующий конфиг он **бэкапит**, не затирает.

---

## 5. Шаг «периметр» (безопасность — не пропускать)
Установщик спросит каталоги, которые агент **не должен трогать** (прод, секреты).
Это физический барьер: агент не дотянется до них, даже если захочет.
- Есть боевой стек/секреты — назови их каталоги через пробел.
- На чистой локалке нечего закрывать — нажми Enter, пропусти.

---

## 6. Активация и проверка
```
source ~/.bashrc
opencode agent list
```
В списке должен быть **Vosmenog**. Затем:
```
opencode
```
Нажми **Tab** — переключаются режимы: `build`, `plan`, **Vosmenog**.
Выбери Vosmenog и спроси: «процитируй свои три стопа и скажи, кто ты по контракту».
Ответ про «исполнитель, не архитектор / работаю по TASK / прод не трогаю» = метод доехал.

> Не появился Vosmenog или права игнорятся → фронтматтер не подхватился. Проверь
> `~/.config/opencode/agents/Vosmenog.md` и версию opencode. Сообщи — разберём.

---

## 7. Лаунчер проектов
`go` заводит проект и запускает opencode прямо в его папке (чтобы подхватились слоты
`<проект>/AGENTS.md`). Открой `~/.bashrc`, найди функцию `go`, добавь свой проект:
```bash
go() {
  case "$1" in
    myproj) cd ~/path/to/myproj && opencode ;;
    *) echo "go: неизвестный проект '$1'" ; return 1 ;;
  esac
}
```
`source ~/.bashrc` — и `go myproj` готов.

---

## 8. Новый проект под Восю
1. В папке проекта создай `AGENTS.md` из шаблона `~/agent-os/project-slots.template.md`,
   заполни слоты (стек, пути, патч, билд, git-состояние, гейты).
2. Переключись на режим Vosmenog (Tab).
3. Работай циклом: **обсуждение → агент пишет EPIC → PBR → go → задачи → resolve.**
   Подробно — `~/agent-os/ONBOARDING.md`.

---

## 9. Первый эпик (обкатка)
Возьми маленькую реальную задачу. В режиме Vosmenog:
- опиши цель, обсудите развилки;
- агент оформит EPIC (DoR/DoD/AC) — проверь, что развилки закрыты;
- дай go, агент раздробит на задачи и исполнит под подтверждение;
- в конце — resolve.

Если на каком-то шаге агент пошёл вперёд без ТЗ или полез не туда — это сигнал,
зафиксируй (BUG дистрибутива) и поправь конфиг/метод.

---

## 10. Серверный сценарий (надстройка, позже)
Всё то же самое, но opencode и проекты живут на VPS, а ты подключаешься:
- с компа — VSCode «Remote-SSH» (терминал на сервере);
- с телефона — Termius по SSH.
Установщик и структура идентичны — меняется только «где запускаешь» и как заливаешь
файлы (git clone прямо на сервере). Аренда VPS и базовая настройка — отдельный раздел,
добавляется после того, как локальный сценарий обкатан.
