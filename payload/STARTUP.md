# Memory System — Boot Instructions

## Принцип

Я — оперативная память сервера. Между сессиями ничего не помню.
journal.md + archive — внешняя память. Читаю при старте, пишу при завершении.

## Старт сессии

1. Прочитай `journal.md` — что было в последних сессиях.
2. Не хватает контекста — читай `archive/YYYY-MM-DD_session.md`.
3. journal.md ≤ 100 строк. Больше — срежь хвост.

## Завершение сессии

Перед прощанием — рефлексия:

1. Создай `archive/YYYY-MM-DD_session.md` с полным саммари.
2. Добавь одну строку в начало `journal.md`.
3. Проверь длину journal.md — > 100 строк → срежь старые.

## Связь с эпиками (важно)

Resolve эпиков живёт в проекте (`docs/epics/`), НЕ здесь — это память проекта.
Но если сессия закрыла/сдвинула эпик — индексной строкой в journal со ссылкой:

```
2026-06-27 EPIC-007 resolved: загрузка файлов → карточки (см. <project>/docs/epics) (a1b2c3d)
```

journal — общий индекс сессий; детали эпика — в его файле в проекте.

## Сессии «владелец + Вося-в-чате»

Если работу вёл владелец через внешний ревью-канал (архитектура, правка método,
сборка фреймворка), а не агент-на-сервере — journal сам не обновится.
Такие сессии journalируем вручную: владелец просит записать строку. Не теряем их.

## Формат саммари в archive/

```markdown
# YYYY-MM-DD — Название сессии

**Задача:** что делали
**Эпик:** EPIC-NNN (если был)
**Ветка:** <ветка>
**Коммит:** хеш

## Что сделано
- ключевые действия

## Ключевые решения
- архитектурные/продуктовые

## Изменённые файлы
- путь — что сделано

## Статус
- завершено / осталось (todo) / отложено

## Рефлексия
- что пошло не так
- что учесть дальше
```

## Формат строки journal.md

```
2026-06-22 docs: консолидация структуры docs/ (00ef88af)
```

Одна строка: дата, кратко, коммит.

## Триггеры рефлексии

- «пока», «завершаем», «на сегодня всё»
- сделан коммит, нет следующей задачи
- явная просьба «напиши саммари»

## Правила

- Не коммитить `~/.config/opencode/` в репозиторий.
- archive/ — только append, никогда не удалять.
- journal.md — только индекс, одна строка на сессию.
