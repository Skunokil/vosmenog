# Vosmenog — фреймворк дисциплинированного агента для opencode

Восилий (Vosmenog) — режим строгого исполнения для [opencode](https://opencode.ai):
агент-исполнитель по детальному ТЗ, который **не уходит в самодеятельность**.
Не архитектор, не трогает прод, документирует работу эпиками (DoR/DoD/AC), эскалирует
развилки вместо того, чтобы решать их догадкой.

## Что внутри

- **METHOD.md** — ядро метода (always-context).
- **agents/Vosmenog.md** — таб-режим с урезанными правами (барьер периметра).
- **EPIC / TASK / BUG** — шаблоны документирования.
- **STARTUP.md** — протокол памяти между сессиями.
- **setup.sh** — установщик (раскладка, конфиг, периметр, лаунчер).

## Установка

```bash
git clone https://github.com/Skunokil/vosmenog.git
cd vosmenog
chmod +x setup.sh
./setup.sh
```

Подробно, со всеми граблями — `GUIDE.md`. Для установки с ИИ-наставником — кинь
`INSTALL_PROMPT.md` первым сообщением в чат с ассистентом, приложив папку.

## Зачем это нужно

Дай агенту большую цель без точного задания — и он начнёт додумывать. Выберет
технологии за вас, полезет менять то, что трогать не просили, развернёт что-нибудь
на боевом сервере «чтобы работало». Не по злому умыслу — просто там, где нет
чёткого ТЗ, модель заполняет пустоту тем, что придумает сама. И обычные правила
в промпте («не трогай прод», «спрашивай перед важным») под нагрузкой забываются —
агент их прочитал, кивнул и всё равно сделал по-своему.

Vosmenog решает это не уговорами, а **устройством**:

- **Руки отняты, а не связаны просьбой.** Доступ к боевым каталогам и опасным
  командам закрыт на уровне движка opencode. Агент не «обещает не трогать прод» —
  он физически до него не дотянется.
- **Без задания — нет работы.** Голая цель «сделай Х» не запускает агента. Сначала
  оформляется документ (что делаем, как проверим, какие решения приняты) — и только
  потом код.
- **Развилки не решает молча.** Встретил выбор без ответа — останавливается и
  выносит человеку, а не угадывает.

Дисциплина держится конфигурацией и структурой, а не доброй волей модели. Поэтому
работает даже на слабых и быстрых моделях, которые обычные текстовые правила
игнорируют.
